# AI 对话窗口集成规范

## 1. 概述

在日程管理器侧边增加 AI 对话窗口，将 AI 一句话创建日程功能整合到对话中，提供更灵活的交互方式。

## 2. 功能需求

### 2.1 侧边 AI 对话窗口

**位置和布局：**
- 在主界面右侧增加一个可展开/收起的侧边栏
- 侧边栏宽度约 350-400px
- 默认可以是收起状态，点击右侧按钮展开
- 包含对话历史区域、输入框和控制选项

**组件：**
- 标题栏：显示 "🤖 AI 助手"
- 对话历史显示区域（滚动）
- 上下文选项区域（复选框）
- 消息输入框
- 发送按钮

### 2.2 对话模式

**默认对话模式：**
- 不勾选"执行命令"复选框时，AI 仅进行普通对话
- 可以询问日程建议、时间管理技巧等
- 不自动修改日程数据

**命令执行模式（勾选"执行命令"）：**
- 勾选后，AI 可以解析用户输入并执行日程操作
- 支持的命令包括：
  - 添加日程
  - 删除日程
  - 修改日程
  - 标记完成
  - 查询日程

### 2.3 上下文选项

**可配置的上下文：**
- [ ] 包含未完成日程（默认勾选）
- [ ] 包含未到期日程
- [ ] 包含待办日程
- [ ] 包含已完成日程（默认不勾选）

**上下文传递：**
- 勾选的选项对应的日程数据会作为上下文传递给 AI
- 格式化为易读的文本格式
- 帮助 AI 更好地理解当前日程状态

### 2.4 对话历史

**显示格式：**
- 用户消息：靠右对齐，蓝色背景
- AI 回复：靠左对齐，灰色背景
- 时间戳显示
- 支持滚动查看历史

**持久化：**
- 对话历史保存在内存中
- 关闭窗口时清空
- 可以添加"清空历史"按钮

## 4. 技术实现

### 4.1 新增组件

**AIChatWidget 类：**
- 继承自 QWidget
- 包含所有对话窗口组件
- 处理对话逻辑

**AIChatMessage 类：**
- 表示单条消息
- 包含角色（用户/AI）、内容、时间戳

### 4.2 修改现有组件

**ScheduleWindow 类：**
- 在 init_ui 中添加 AI 对话侧边栏
- 添加展开/收起按钮
- 连接相关信号和槽

### 4.3 AI 交互逻辑

**两种模式：**
1. **对话模式**：普通聊天，使用 get_suggestion 风格的 API
2. **命令模式**：执行日程操作，使用 run_command 风格的 API

**上下文构建：**
```python
def build_context(self):
    context = []
    if self.include_unfinished.isChecked():
        # 添加未完成日程
    if self.include_not_expired.isChecked():
        # 添加未到期日程
    if self.include_todo.isChecked():
        # 添加待办日程
    if self.include_completed.isChecked():
        # 添加已完成日程
    return "\n".join(context)
```
